خطة تطوير البرمجيات الشاملة والمشتركة

منصة SaaS ذكية متعددة الشركات لإدارة علاقات العملاء، المبيعات، الفواتير، والمهام مع الذكاء الاصطناعي

الفريق: 6 أفراد (تطوير مشترك) المدة المستهدفة: 3 أسابيع (تطوير مكثف) التقنيات: ASP.NET Core + Angular + AI

1. الفلسفة العامة للخطة: "الجميع يعمل على كل شيء"

بما أن الفريق يفضل عدم حصر أي فرد في مسمى وظيفي واحد (مثل واجهات فقط أو قاعدة بيانات فقط)، بل يرغب الجميع في المشاركة في دورة التطوير كاملة (Full-Stack)، فقد تم تصميم هذه الخطة باتباع منهجية تطوير الميزات المتكاملة (Feature-Driven & Cross-Functional Development).

كيف سيتم تنظيم العمل؟ سنقسم الـ 6 أفراد إلى 3 مجموعات ثنائية (Pairs) تتغير تركيبتها دورياً. كل مجموعة ثنائية تتبنى ميزة كاملة (Feature) وتقوم ببرمجتها بالكامل: بدءاً من تصميم جدول قاعدة البيانات، مروراً بكتابة الـ API في الـ Backend، ووصولاً لبناء الواجهات في الـ Frontend وربطها بالذكاء الاصطناعي واختبارها.

ميزة هذا الأسلوب: يضمن زيادة المهارات الفنية للجميع بالتساوي في الـ Backend و الـ Frontend و الـ AI، ويسرع وتيرة العمل ويسد الفجوات البرمجية، بالإضافة إلى ضمان عمل واجهات متناسقة مع منطق البرمجة الخلفية بشكل فوري.

2. مسار تدفق الأدوار والمستخدمين في النظام (User Flows)

قبل كتابة الكود، يجب فهم كيف سيعمل النظام من لحظة دخول المستخدم وحتى انتهائه. تم تقسيم الصلاحيات والأدوار كالتالي:

أ. مسار مالك الشركة (Owner - مستأجر النظام)

  1. التسجيل والتأسيس: يدخل إلى صفحة التسجيل ➔ يملأ بيانات شركته (أول مستأجر) وحسابه الشخصي كمالك للشركة.
  2. عزل البيانات: يقوم النظام بتوليد معرف فريد TenantId وربطه بجميع بيانات شركته لضمان الخصوصية التامة.
  3. إعداد الفريق: يذهب إلى لوحة تحكم الإدارة ➔ يرسل دعوات إلكترونية للموظفين ويسند إليهم أدوارهم (Admin, Sales, Accountant, Support).
  4. الرقابة: يراقب الرسوم البيانية اللحظية (SignalR) لأرباح الشركة، المهام المفتوحة، والفواتير المتأخرة، ويسأل المساعد الذكي عن إحصائيات شركته.

ب. مسار مدير العمليات (Admin)

  1. القبول والدخول: يقبل دعوة المالك ➔ يسجل كلمة مروره ➔ يسجل الدخول للنظام وتظهر له البيانات المفلترة لشركته فقط.
  2. إدارة العمل والصفقات: يراجع مسار الصفقات المفتوحة (Deals Pipeline) وأعمال موظفي المبيعات.
  3. رفع الاجتماعات (Meeting Intelligence): يقوم برفع تسجيل اجتماع الفريق ➔ ينتظر معالجة الصوت (Whisper) والتلخيص (AI) ➔ يظهر له ملخص الاجتماع وتاسكات مقترحة ➔ يراجعها ويضغط "Approve & Create" لتتحول فوراً لتاسكات مضافة ومسندة في النظام.

ج. مسار مندوب المبيعات (Sales)

  1. إدارة علاقات العملاء CRM: يضيف عميلاً جديداً ➔ يسجل الملاحظات ويرفع المرفقات الخاصة بالعميل.
  2. المبيعات والصفقات: ينشئ صفقة جديدة للعميل ➔ يتحكم في حالتها بسحبها وإفلاتها (Drag & Drop) في لوحة الصفقات التفاعلية.
  3. تحليل العقود (Contract Intelligence): يرفع ملف العقد المتفق عليه (PDF/Word) ➔ يقرأ تحليل الذكاء الاصطناعي الملون (أحمر للمخاطر، برتقالي للتحذيرات، أزرق للمعلومات) للتأكد من سلامة البنود قبل توقيعها وحفظها بملف العميل.
  4. المهام اليومية: يتابع المهام الموكلة إليه (المستخرجة من الاجتماعات مثلاً) ويقوم بتغيير حالتها (Todo ➔ In Progress ➔ Done).

د. مسار المحاسب (Accountant)

  1. توليد الفواتير: يسجل الدخول ➔ ينشئ فاتورة جديدة لصفقة مبيعات رابحة (Won Deal).
  2. التصدير والطباعة: يقوم بتصدير الفاتورة كملف PDF احترافي منسق (QuestPDF) لإرسالها للعميل.
  3. متابعة التحصيل: يغير حالة الفاتورة لـ (Paid) عند السداد. الفواتير غير المدفوعة التي تتجاوز تاريخ استحقاقها تتحول تلقائياً إلى (Overdue) عبر وظيفة خلفية (Hangfire).

هـ. مسار الدعم الفني (Support)

  1. المراجعة والمساعدة: يسجل الدخول بصلاحيات مقيدة تسمح له بالبحث في العملاء ومراجعة تفاصيلهم وكتابة ملاحظات (Notes) لحل مشاكلهم، دون أي إمكانية للوصول للفواتير أو الصفقات المالية.

3. خطة العمل التفصيلية (3 أسابيع - تطوير كامل متوازي)

تم تقسيم المهام أسبوعياً بحيث تعمل المجموعات الثلاث في نفس الوقت على مهام متوازية ومتكاملة لضمان اكتمال المنتج في 21 يوماً.

 تأسيس واجهة المستخدم (Angular Shell)

الأسبوع الثاني: موديولات العمليات موديول إدارة العملاء CRM

  • تطوير شاشة مسار الصفقات التفاعلية (Kanban Deals Board).
  • ربط المبيعات بعلاقات مع العملاء وقيم الصفقات المتوقعة.
 موديول المهام والـ Live Dashboard
  • تطوير موديول المهام (Tasks CRUD) وإسنادها للمستخدمين وتحديث حالتها.
  • برمجة الـ SignalR Hub لإرسال تحديثات لوحة القيادة اللحظية.
  • بناء شاشات الرسوم البيانية للـ KPIs والمبيعات المباشرة.
 موديول الفواتير والعمليات الخلفية
  • تطوير الـ APIs للفواتير وحالاتها وتوليد رقم الفاتورة التلقائي.
  • برمجة تصدير الفواتير كـ PDF احترافي (QuestPDF).
  • إعداد جدولة المهام بـ Hangfire لتحديث الحالات تلقائياً وإرسال إيميلات التذكير.
الأسبوع الثالث: الذكاء الاصطناعي والصقل ميزة ذكاء المستندات وتحليل العقود
  • برمجة استخراج النصوص من ملفات Word/PDF (PdfPig + OpenXML).
  • إعداد برومبت الذكاء الاصطناعي للـ LLM لاستخراج التلخيص والأخطار والتحذيرات كـ JSON.
  • شاشات رفع وعرض العقود وتحليلها بالألوان (Danger, Warning, Info).
 ميزة ذكاء الاجتماعات وتفريغ الفيديو
  • إدماج Whisper لتفريغ ملفات الفيديو والصوت المرفوعة إلى نصوص.
  • كتابة برومبت استخراج القرارات والتاسكات المقترحة من النص المفرغ.
  • شاشات رفع الفيديو وعرض المهام المقترحة وزر الحفظ التلقائي في موديول المهام.
 المساعد الذكي ونشر النظام (DevOps)
  • تطوير الـ AI Assistant في الشات باستخدام الـ Tool Calling لقراءة بيانات الشركة بشكل آمن.
  • تأمين المشروع من ثغرات عزل البيانات واختبار ثغرات الـ IDOR.
  • إعداد Docker وحزم المشروع والـ deployment على سيرفر حي للعرض.

4. ميزات إضافية ذات أثر هندسي وتجاري حقيقي

لضمان تميز المشروع وحصوله على الدرجات النهائية وتجاوز لجان التقييم بالامتياز، نقترح إضافة الميزتين التاليتين:

✦ التنبيهات الاستباقية الذكية والربط بالواتساب (AI WhatsApp Alerts)

بدلاً من تصفح المحاسب للفواتير المتأخرة، يقوم خادم Hangfire بالخلفية بفحص الفواتير يومياً. الفواتير التي يقترب موعد استحقاقها تُرسل للـ LLM الذي يصيغ رسالة تنبيه مخصصة ولطيفة باللغة العربية للعميل، ويقوم النظام بإرسالها تلقائياً له كرسالة WhatsApp عبر API مثل (Twilio/Ultramsg) أو كـ Email.

✦ البحث الدلالي والـ RAG داخل العقود (Semantic Document Search)

عند رفع عشرات العقود، يستطيع المستخدم كتابة سؤال مثل: "ما هي غرامة التأخير في عقد شركة النور؟". نستخدم إضافة pgvector لحفظ نصوص العقود كـ Embeddings في قاعدة PostgreSQL، حيث يقوم النظام بالبحث الدلالي عن المعنى، واستخلاص الفقرة المعنية وعرضها ملخصة ومباشرة للمستخدم.

توصية ذهبية ليوم العرض (Demo Day): لتجنب أي بطء قد يسببه تشغيل نماذج الـ AI محلياً على أجهزة معالجاتها متوسطة (CPU)، صممنا طبقة الـ AI بنمط Provider Pattern (حيث يمكن التبديل بين تشغيل نموذج محلي Ollama أو نموذج سحابي API مجاني كـ Gemini Free Tier بسطر واحد في ملف الإعدادات). يفضل بشدة استخدام ملفات فيديو قصيرة (2-4 دقائق) وعقود قصيرة (2-5 صفحات) أثناء العرض أمام اللجنة لضمان سرعة الاستجابة الفورية للميزات واستعراض الذكاء الاصطناعي بشكل مقنع ومبهر.
